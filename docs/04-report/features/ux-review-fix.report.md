## 3. 해결된 문제

### 3.1 사이드바 accountId 자동 추출

| 항목 | 내용 |
|------|------|
| **문제** | `/accounts/[accountId]` 페이지에서 사이드바에 계정별 메뉴가 표시되지 않음 |
| **원인** | `Sidebar` 컴포넌트에 `accountId` prop이 전달되지 않음 |
| **해결** | URL 경로에서 accountId 자동 추출 로직 추가 |

**수정 코드** (`src/components/layout/sidebar.tsx:68`):
```tsx
const extractedAccountId = accountId || pathname.match(/\/accounts\/([^\/]+)/)?.[1];

const navItems = extractedAccountId
  ? getAccountNavItems(extractedAccountId)
  : mainNavItems;
```

### 3.2 대시보드 "자세히 보기" 버튼 연결

| 항목 | 내용 |
|------|------|
| **문제** | 긴급 알림의 "자세히 보기" 버튼 클릭 시 동작 없음 |
| **원인** | Button에 onClick 또는 Link 연결 누락 |
| **해결** | Link 컴포넌트로 감싸서 인사이트 페이지로 연결 |

**수정 코드** (`src/app/(dashboard)/accounts/[accountId]/page.tsx:456-460`):
```tsx
<Link href={`/accounts/${accountId}/insights?severity=CRITICAL`}>
  <Button variant="destructive" size="sm">
    자세히 보기
  </Button>
</Link>
```

## 4. 전체 페이지 연결 검증 결과

```
✅ 홈 (/)
   └─ Get Started → /accounts

✅ 계정 목록 (/accounts)
   ├─ 계정 연동 → /api/auth/tiktok
   └─ 계정 카드 → /accounts/{id}

✅ 대시보드 (/accounts/{id})
   ├─ 새로고침 → API refetch
   ├─ 날짜 선택 → URL param
   ├─ 이전 기간 비교 → 차트 토글
   ├─ 긴급 알림 → /accounts/{id}/insights?severity=CRITICAL
   ├─ 최근 인사이트 → /accounts/{id}/insights
   ├─ 대기 전략 → /accounts/{id}/strategies
   └─ 전략 수락/거절 → PUT API

✅ 소재 분석 (/accounts/{id}/creatives)
   ├─ 소재 동기화 → POST API
   ├─ 정렬/뷰 모드 → 상태 업데이트
   └─ 인사이트 보기 → /accounts/{id}/creatives/{creativeId}

✅ 소재 상세 (/accounts/{id}/creatives/{creativeId})
   ├─ 돌아가기 → router.back()
   ├─ 인사이트 생성 → POST API
   ├─ 전략 생성 → POST API
   └─ 전략 수락/거절 → POST API

✅ AI 인사이트 (/accounts/{id}/insights)
   ├─ 인사이트 생성 → POST API
   ├─ 필터 → URL param
   └─ 읽음 처리 → PUT API

✅ AI 전략 (/accounts/{id}/strategies)
   ├─ 전략 생성 → POST API
   ├─ 필터/탭 → URL param
   └─ 수락/거절/시작/완료 → PUT API

✅ 리포트 (/accounts/{id}/reports)
   ├─ 리포트 생성 → POST API
   ├─ 다운로드 → GET API
   └─ 삭제 → DELETE API

✅ 알림 (/notifications)
   ├─ 모두 읽음 → PUT API
   ├─ 보기 → 링크 이동
   └─ 삭제 → DELETE API
```
